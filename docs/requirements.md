# GameVault 需求分析

## 1. 系统概述

GameVault 是一个面向游戏玩家的游戏愿望单与游玩进度分析系统。用户可以记录想玩的游戏、已购买游戏、正在玩的游戏和已通关游戏，并对平台、类型、标签、价格、评分和游玩时长进行管理和统计。

## 2. 用户需求

普通用户需要完成以下操作：

- 注册账号并登录系统。
- 维护个人游戏库。
- 添加、修改、删除游戏记录。
- 管理游戏状态、优先级、购买价格、游玩时长和个人评价。
- 按名称搜索游戏。
- 查看平台分布、类型分布、游戏数量、愿望单数量、通关数量和总消费金额。

## 3. 功能需求

### 3.1 用户认证

- 用户可以注册账号。
- 用户可以使用用户名和密码登录。
- 系统使用 JWT 验证用户身份。
- 未登录用户不能访问个人游戏库和统计接口。

### 3.2 游戏管理

- 用户可以新增游戏基础信息。
- 用户可以查询游戏列表和游戏详情。
- 用户可以修改或删除游戏。
- 游戏可以关联多个平台、类型和标签。

### 3.3 个人游戏库

- 用户可以将游戏加入个人游戏库。
- 用户可以设置状态、优先级、价格、评分、时长和评价。
- 用户可以删除自己的游戏库记录。

### 3.4 分类统计

- 系统展示平台、类型和标签。
- 系统统计不同平台和不同类型下的游戏数量。
- 系统展示总游戏数、愿望单数量、正在玩数量、已通关数量、总消费和总时长。

## 4. 非功能需求

- 系统应支持 Docker Compose 一键启动。
- 页面应美观，适合课堂演示。
- 数据库使用 MySQL，满足课程要求。
- 后端接口应提供 Swagger 文档。
