# GameVault 答辩讲稿

大家好，我们小组的项目是 GameVault 游戏愿望单与游玩进度分析系统。

本系统面向游戏玩家，解决的问题是游戏收藏分散、愿望单难以管理、游玩进度和消费数据缺少统计的问题。系统使用 MySQL 保存核心业务数据，后端使用 FastAPI 提供接口，前端使用 React 展示页面，并通过 Docker Compose 实现一键启动。

数据库方面，我们设计了用户表、游戏表、个人游戏库表、游玩日志表、平台表、类型表、标签表以及三张多对多关联表。用户和个人游戏库是一对多关系，游戏和平台、类型、标签是多对多关系，个人游戏库和游玩日志是一对多关系。这样的设计能够比较完整地体现实体完整性和参照完整性。

功能方面，系统支持用户注册登录、游戏新增编辑删除、游戏详情查看、愿望单管理、状态切换、游玩日志维护、平台类型标签管理、个人资料修改和统计分析。统计分析包括平台分布、类型分布、状态分布、月度新增趋势、月度游玩时长趋势、评分榜和游玩时长榜。

演示时，我们会先登录系统，然后展示首页仪表盘，再新增一款游戏，设置平台、类型和标签。接着进入游戏库查看详情、修改状态，并添加一条游玩日志。最后展示统计分析页面和 Adminer 中的数据库表数据变化。

测试方面，我们为用户认证、游戏管理、分类管理、游玩日志、统计接口和权限校验编写了测试用例，目标是后端测试覆盖率达到 90% 以上。

总结来说，GameVault 完成了一个完整的数据库 Web 应用系统，满足课程对需求分析、数据库设计、前后端开发、数据增删改查、测试和现场演示的要求。
